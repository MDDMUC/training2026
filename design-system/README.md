# Training 2026 — Design System

## Token architecture (three tiers)

```
primitive  →  semantic   →  component
(raw value)   (intent)      (override)
gray.900      surface.fg    button.label
```

- **Primitive tokens** are raw values. Color hexes, exact pixel sizes, font weights as numbers. They are context-free.
- **Semantic tokens** name a primitive by *intent*. `color.fg.default` rather than `gray.950`. Everything in components should reference these.
- **Component tokens** override a semantic when a single component needs a special value. Use sparingly.

Components **never** reference primitive tokens directly. This is the single rule that keeps the system small and changeable.

## Stack-agnostic

Tokens are authored in W3C Design Tokens Format (DTCG 2025.10) so they survive any framework choice — React, Svelte, vanilla, or HTMX. The CSS file is generated by hand for now; once we pick a framework we can wire Style Dictionary or equivalent.
